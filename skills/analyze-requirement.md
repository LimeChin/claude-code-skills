# 需求分析 Skill

分析需求文档，自动扫描代码库和数据库，生成实现方案。

## 用法

```
/analyze-requirement <需求文档路径> [代码记录路径]
```

## 执行步骤

### 1. 读取输入文件

- 读取需求文档（必须）
- 读取代码记录（如果有）
- 确认当前工作的微服务项目

### 2. 加载数据库配置

从 `D:\winchannel\db-config.local.yml` 读取数据库连接配置。配置格式：

```yaml
projects:
  vinda-nacos-dev:
    host: 192.168.x.x
    port: 3306
    database: base_oms
    username: root
    password: xxx
  ipsen-dev:
    host: ...
```

如果配置文件不存在，询问用户是否需要创建模板或跳过数据库查询。

### 3. 扫描代码库

根据需求内容，自动扫描：

- **关键词匹配**：从需求中提取关键词，搜索相关代码
- **实体类**：查找涉及的数据库表对应的 Entity
- **Service/Controller**：查找相关的业务逻辑和接口
- **Mapper**：查找相关的数据库操作

输出：
- 涉及的文件清单
- 文件路径和简要说明

### 4. 查询数据库表结构

如果数据库配置可用，查询涉及的表结构：

- 表名、字段名、字段类型
- 主键、索引
- 表之间的关联关系

### 5. 分析依赖关系

- 分析代码调用链
- 标注改动影响范围

### 6. 生成实现方案

对每个需求点输出：

```
## 需求点：xxx

### 涉及文件
- 文件路径：说明

### 数据表
- 表名：字段说明

### 实现方案
- 方案 A：xxx（推荐）
- 方案 B：xxx

### 推荐顺序
1. 先做 xxx
2. 再做 xxx
```

### 7. 用户选择

列出所有需求点，让用户选择：
- 选择要实现的需求点
- 选择实现方案

### 8. 执行实现

根据用户选择，逐个实现需求点。

## 注意事项

- 数据库配置文件包含敏感信息，不要提交到 git
- 每次执行前确认当前工作的微服务项目
- 如果扫描结果不准确，可以让用户手动指定关键词
