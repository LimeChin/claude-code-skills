# 需求分析 Skill

分析需求文档，自动扫描代码库和数据库，生成实现方案。

## 用法

```
/analyze-requirement <需求文档路径> [代码记录路径]
```

## 执行步骤

### 1. 读取需求文档

- 读取需求文档（必须）
- 读取代码记录（如果有）
- 确认当前工作的微服务项目

### 2. 判断开发模式

**根据是否有代码记录决定开发模式：**

**模式A：有代码记录 → 跟踪开发**
- 如果提供了代码记录路径且文件存在，读取代码记录
- 根据代码记录中已完成的进度，继续后续开发
- 不需要重新选择需求点，按代码记录中的顺序继续

**模式B：无代码记录 → 全新开发**
- 解析需求文档，提取所有功能模块和需求点
- 使用 AskUserQuestion 工具让用户选择要开发的需求点

### 3. 解析需求点（仅模式B）

分析需求文档，提取所有功能模块和需求点，以列表形式展示：

```
## 需求文档分析结果

### 功能模块列表
1. [模块名] 功能描述
   - 需求点 2.1：xxx
   - 需求点 2.2：xxx
2. [模块名] 功能描述
   - 需求点 3.1：xxx
   ...
```

### 4. 用户选择需求点（仅模式B）

**使用 AskUserQuestion 工具让用户选择**

列出所有需求点，让用户选择：
- 要开发哪个功能模块或需求点
- 可以多选或单选

**等用户确认选择后，再继续后续步骤**

### 5. 加载数据库配置

从 `D:\winchannel\db-config.local.yml` 读取数据库连接配置。配置格式：

```yaml
projects:
  vinda-nacos-dev:
    host: 192.168.x.x
    port: 3306
    database: base_oms
    username: root
    password: xxx
  ipsen-dev:
    host: ...
```

如果配置文件不存在，询问用户是否需要创建模板或跳过数据库查询。

### 6. 扫描代码库

根据用户选择的需求点（模式B）或代码记录中的需求点（模式A），扫描相关代码：

- **关键词匹配**：从需求中提取关键词，搜索相关代码
- **实体类**：查找涉及的数据库表对应的 Entity
- **Service/Controller**：查找相关的业务逻辑和接口
- **Mapper**：查找相关的数据库操作

输出：
- 涉及的文件清单
- 文件路径和简要说明

### 7. 查询数据库表结构

如果数据库配置可用，查询涉及的表结构：

- 表名、字段名、字段类型
- 主键、索引
- 表之间的关联关系

### 8. 分析依赖关系

- 分析代码调用链
- 标注改动影响范围

### 9. 生成实现方案

对用户选择的需求点输出：

```
## 需求点：xxx

### 涉及文件
- 文件路径：说明

### 数据表
- 表名：字段说明

### 实现方案
- 方案 A：xxx（推荐）
- 方案 B：xxx

### 实现步骤
1. 先做 xxx
2. 再做 xxx
```

### 10. 用户确认方案

使用 AskUserQuestion 工具让用户确认：
- 是否同意该实现方案
- 是否需要调整

### 11. 执行实现

用户确认后，开始编码实现。

## 注意事项

- **必须先让用户选择需求点**，不要自作主张开始开发
- 数据库配置文件包含敏感信息，不要提交到 git
- 每次执行前确认当前工作的微服务项目
- 如果扫描结果不准确，可以让用户手动指定关键词

## 代码记录文件格式

代码记录用于跟踪开发进度，格式如下：

```markdown
# 需求编号：2026-0175

## 当前状态
- 当前需求点：3.2.5 添加购物车
- 已完成：3.2.1 ~ 3.2.4

## 需求点开发记录

### 3.2.1 选择下单经销商
- 状态：✅ 已完成
- 涉及文件：xxx
- 提交记录：abc123

### 3.2.2 选择收货地址
- 状态：✅ 已完成
- 涉及文件：xxx
- 提交记录：def456

### 3.2.3 产品与价格显示
- 状态：🔄 进行中
- 当前任务：实现特价显示逻辑
```
